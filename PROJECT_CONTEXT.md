# 项目背景

ResumeMatch Studio 是一个在线简历编辑器，面向需要快速打磨简历、未来希望接入 JD 匹配与简历改写能力的求职者。当前版本先把简历模板编辑、排版、暂存、导出 PDF、云端访问做稳定，再逐步接入 JD 匹配系统。

线上地址：https://resumematch-studio.vercel.app

GitHub 仓库：https://github.com/00lang786/resumematch-studio

# 当前目标

先继续打磨简历编辑器体验，保证本地与线上版本稳定可用。当前重点是保持 GitHub 与 Vercel 同步，并为后续 JD 匹配接入做数据结构准备。

# 技术栈

- 前端：Vite、React、TypeScript、Tailwind CSS v4
- 图标：lucide-react
- UI 组件：Figma Make 导出的 shadcn/ui / Radix 组件文件
- 后端：暂无
- 数据库：暂无
- 本地存储：浏览器 localStorage，key 为 `resume-template-draft`
- 部署：Vercel

# 核心功能

- A4 风格简历预览与编辑
- 个人信息、个人总结、工作经历、教育经历、专业技能编辑
- 模块标题改名、显示/隐藏、上下排序
- 工作经历上下排序
- 职责描述上下排序
- 本地暂存草稿
- 导出 JSON / 导入 JSON，用于长期保存与跨设备迁移
- 打印 / 导出 PDF
- 个人总结和职责描述支持轻量格式：
  - `**加粗**`
  - `{blue:蓝色}`
  - `{blue:**蓝色加粗**}`
  - `{gray:灰色}`
  - `{red:红色}`
- 格式快捷按钮：B、蓝色、蓝色+B、灰色、红色
- 全局字体使用微软雅黑
- 柔和蓝色视觉方案，主色为 `#1f5f99`

# 已知问题

- `src/app/App.tsx` 过大，业务逻辑和 UI 混在一个文件里
- localStorage 只保存在当前浏览器，换浏览器/电脑或清缓存会丢失
- 浏览器打印 PDF 的页眉页脚需要用户手动关闭
- 打印颜色在不同浏览器中仍可能有差异
- shadcn/ui 导出组件很多，但大部分暂未使用
- 当前没有云端数据库，不支持多设备同步
- 格式快捷按钮仍是轻量标记方案，不是真正富文本

# 重要决策记录

- 项目名称定为 ResumeMatch Studio。
- 当前先做轻量简历编辑器，不急于接复杂后端。
- 暂存采用 localStorage，适合本机继续编辑，不作为长期可靠存储。
- 长期保存与跨设备迁移采用 JSON 导出 / 导入作为第一版方案。
- 默认模板必须使用安全示例内容，避免真实简历信息外泄。
- 富文本采用轻量标记语法，而不是引入复杂富文本编辑器。
- 为降低输入成本，增加格式快捷按钮，底层仍保存纯文本标记。
- 排序采用上移/下移按钮，不做拖拽作为第一版。
- 简历视觉风格保持正式、克制，正文使用黑色，字体使用微软雅黑。
- 模块标题使用灰色背景，标题黑色加粗。
- 蓝色主色使用较柔和的 `#1f5f99`。
- 不允许虚构用户经历、公司、岗位或数据。
- 未使用的个人图片素材不上传 GitHub。
